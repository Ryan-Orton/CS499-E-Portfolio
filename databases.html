Databases Enhancement

Artifact Description

This artifact is a mobile Weight Tracker application originally created in CS 360, Mobile Architecture and Programming. I selected this artifact for my ePortfolio because it provides a strong foundation for demonstrating multiple areas of computer science, including software design and engineering, algorithms and data structures, and databases.

Why I Selected This Artifact

For the databases category, this artifact was especially appropriate because it already relied on SQLite for storing user accounts, goal weights, and weight history, which gave me a practical starting point for improving database structure, data integrity, and security.

Enhancement Performed

In the original version of the application, user credentials were stored in plain text, weight entries were not tied to a specific user, and goal weights were stored globally instead of per account. To enhance the artifact, I redesigned the database schema so that each user has a unique identifier, each weight entry is associated with a specific user through a foreign key, and each goal weight is tied to one user record. I also improved security by hashing passwords before storing them in the database instead of saving raw passwords directly.

Reflection

Through this enhancement process, I learned that database design is not only about storing data, but about organizing relationships in a way that protects accuracy, privacy, and long-term maintainability. One challenge I faced was that changing the schema also required me to update the rest of the application logic.
