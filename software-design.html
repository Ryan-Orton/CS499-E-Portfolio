Software Design and Engineering Enhancement

Artifact Description

The artifact I selected for this enhancement is my Weight Tracker mobile application, which I originally created in CS 360 Mobile Architecture and Programming. This application allows users to create an account, log in, set a goal weight, enter weight records, store those entries in a local SQLite database, and receive an SMS notification when a goal weight is reached.

Why I Selected This Artifact

I selected this artifact for my ePortfolio because it demonstrates mobile development, user interaction, local data storage, and practical application design. It also provided a clear opportunity for meaningful enhancement in the software design and engineering category.

Enhancement Performed

To enhance the artifact, I implemented the missing update feature in the weight-entry screen. Users can now long-press an entry, choose the update option, edit the weight value in a dialog, and save the change back to the database. I also improved input validation so the application handles blank, non-numeric, and unrealistic values more safely. In addition, I refactored the WeightFragment and CreateAccountFragment classes by moving repeated logic into smaller helper methods.

Reflection

This enhancement helped me better understand how maintainability, usability, and defensive programming all contribute to stronger software. One challenge I faced was deciding how to make the enhancement meaningful enough for the milestone without changing the entire architecture of the application.
