Professional Self-Assessment

Completing the coursework in the Computer Science program and developing this ePortfolio has helped me better understand my strengths, refine my professional goals, and prepare to enter the field of software engineering with greater confidence. Throughout the program, I have developed technical skills in software design, programming, databases, security, mobile development, and problem-solving. Just as importantly, I have learned that being successful in computer science requires more than writing code. It also requires clear communication, thoughtful design decisions, collaboration, adaptability, and a strong awareness of how technical work affects users and organizations. This ePortfolio reflects that growth by showing not only what I have built, but also how I evaluate, improve, and present technical work in a professional setting.

One of the most valuable things I gained from the program was a stronger understanding of my own professional direction. When I first began the program, I had a general interest in technology and software, but over time my career goals became more focused. Through coursework and project work, I found that I am especially interested in software engineering and application development, particularly the process of taking an idea, turning it into a working system, and then improving that system over time. This portfolio supports that direction by highlighting my ability to analyze an existing application, identify weaknesses, design improvements, and implement meaningful enhancements that make the software more useful and more maintainable.

The Computer Science program also helped me better understand the importance of collaboration in software development. Even when coursework was completed individually, many assignments reflected the kinds of collaborative practices that are common in professional environments. Code review, design documentation, requirements analysis, and structured testing all reinforced the idea that software is rarely built in isolation. In professional settings, developers must produce code that other people can read, maintain, and evaluate. My code review work in CS 499 strengthened that understanding because it required me to explain the current state of my artifact, identify its limitations, and justify enhancement decisions in a way that would make sense to peers, instructors, or managers. That process helped me think more intentionally about how collaboration and communication support better technical decision-making.

Communication with stakeholders is another area in which I have grown throughout the program. A computer science professional must be able to adapt technical communication to different audiences, whether speaking with developers, project managers, end users, or nontechnical stakeholders. Across the program, I practiced explaining technical work through narratives, project documentation, presentations, and written reflections. In this capstone, that growth is reflected in both the code review and the narratives that accompany my artifact enhancements. These materials required me to clearly explain what the software does, what problems I identified, what changes I made, and why those changes matter.

The area of data structures and algorithms has also been an important part of my growth. Through the program, I learned that solving a problem effectively is not just about making something work, but about choosing an approach that is logical, maintainable, and appropriate for the situation. In my portfolio artifact, the algorithms and data structures enhancement focuses on making better use of stored weight-entry data. Rather than only storing and displaying entries, I expanded the application to process a list of weight records and generate useful summary information such as starting weight, current weight, total change, average weight, distance from goal, and trend.

My growth in software engineering and database design is reflected strongly throughout this portfolio. The main artifact in my ePortfolio is my Weight Tracker mobile application from CS 360 Mobile Architecture and Programming. I selected this artifact because it provided opportunities to demonstrate improvement in all three required areas: software design and engineering, algorithms and data structures, and databases.

The database portion of my learning has shown me how important data design is to the success of an application. Working with structured data requires more than just storing values; it involves thinking carefully about relationships, integrity, retrieval, and security. In the Weight Tracker artifact, the database enhancement focuses on redesigning the SQLite structure so that user accounts, weight entries, and goal values are properly connected and handled in a more secure way.

Security has been another major theme across the program and one that continues to shape how I think about software development. I have learned that secure design is not something that should be added only at the end of a project. Instead, security needs to be considered from the beginning through validation, careful data handling, reduced assumptions, and awareness of potential vulnerabilities.

This ePortfolio also demonstrates how my artifacts fit together as a whole. Although I am using one main artifact, the Weight Tracker application, the three enhancements show different dimensions of computer science skill. The software design and engineering enhancement demonstrates my ability to improve maintainability, usability, and code organization. The algorithms and data structures enhancement demonstrates my ability to use stored data more intelligently through list processing and summary calculations. The database enhancement demonstrates my ability to improve data structure, integrity, and security.

Overall, the Computer Science program has prepared me not only by helping me build technical skills, but by helping me understand how those skills fit into professional practice. I have grown in programming, design, problem-solving, data management, communication, and security awareness. I have also developed a clearer sense of my career direction and how I want to continue building on what I have learned. This ePortfolio represents that growth and serves as both a reflection on my academic journey and a professional presentation of the skills I am bringing forward into the next stage of my career.
